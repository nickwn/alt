Focus Time Case Study

Team Members: Jack Yang, Michele Murakami

Project Overview


As a part of UCSD’s Interaction Design course, I worked with a team of two other students to research and develop a mobile app to motivate users to be more productive. Our final app, called Focus Time, allowed users to create “focus sessions” with a consequence and reward for finishing or failing to finish their task. I was primarily responsible for implementing the base design of many components of the website in HTML/CSS/JS and designing the calendar view of the website.



A screenshot of the focus sessions page, calendar page, and focus session info page


Problem Statement

As a group of university students, we had all experienced the ways in which distractions could prevent us from doing important classwork. Having an effective method to focus isn’t just important for completing schoolwork, but also for many other aspects of life. Learning a new skill, finishing a book, or doing chores all require time and dedication.
Many students use time management software like Google Calendar to set aside time to study for classes. Unfortunately, even with the time put down, it can be hard for students to stay motivated to do their work.
The question is this: once time is set aside, how can students be motivated to complete their work?

Design Iteration 1: Paper Prototypes

To get a rough idea of what our app should look like, we started with paper prototypes and progressively iterated over our design towards a fully coded real website.

To test this paper prototype, our team conducted a number of qualitative heuristic design evaluations, based on Neilson’s ten design heuristics. After these evaluations, we decided to improve our app in the following ways:

  • Add an about screen to explain to newcomers how the app works
  • Make the functions of input fields more clear. For example, we would have hint text saying “ reward here”, etc
  • Use apple’s ui for time selection to make selecting a time more intuitive
  • Make it clearer to user you can tap on menu by juggling menu focuses
  • Add a way to delete events once they are created to fix mistakenly created events
  • Add buttons like a calendar button for easier and quicker navigation

Design Iteration 3: Web Prototype

We then translated these designs into a website using basic html/css, and eventually added a backend and improved the frontend using handlebars. We initially used a grey color scheme because we had not finalized the look of the app.

To test our app, we had some potential users run through a few basic activities in our app and recorded any potential points for improvement.

  • User #1’s main struggle was the calendar. The dots under each date were supposed to represent whether the event on that day was completed, failed, or pending, but she was confused by what they meant.
  • User #2’s main struggle was deciding on a consequence and reward.
  • User #3 struggled with some of the design decisions in our app, for example the chevron made it seem like the user was supposed to swipe instead of tap.

One of the users had trouble determining the meaning of the dots under each date

To address these concerns, we decided to modify our app in the following ways:

  • Make design of app more clear and engaging
    • Make key for calendar, so user is aware of what the colored dots mean
    • Give suggestions for rewards, consequences, maybe have an example of an event already filled in
  • If possible, allow user to view event when tapping on calendar
  • Add edit button, so user can edit already created sessions
